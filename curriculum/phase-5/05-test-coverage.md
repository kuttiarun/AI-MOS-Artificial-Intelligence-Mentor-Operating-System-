# Test Coverage — Measuring What You've Tested

## Why Coverage Metrics Exist
Test coverage answers: "What percentage of your code is actually exercised by tests?" A codebase with 0% coverage ships bugs blindly. A codebase targeting 80%+ coverage catches most regressions automatically.

## Types of Coverage
| Coverage Type | What It Measures | Example |
|---|---|---|
| Line Coverage | Lines executed | `int x = 5;` was reached |
| Branch Coverage | Each true/false branch | both `if` and `else` executed |
| Method Coverage | Methods called | `findById()` was invoked |
| Statement Coverage | Individual statements | similar to line |

**Branch coverage is more meaningful than line coverage** — a line with `if (x > 0)` has two branches (true/false).

## JaCoCo — The Industry Standard Tool
```xml
<!-- pom.xml — add the plugin -->
<plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <version>0.8.11</version>
    <executions>
        <execution>
            <goals><goal>prepare-agent</goal></goals>
        </execution>
        <execution>
            <id>report</id>
            <phase>test</phase>
            <goals><goal>report</goal></goals>
        </execution>
    </executions>
</plugin>
```

```bash
mvn test                    # runs tests + generates coverage
# Report appears at: target/site/jacoco/index.html
```


## Coverage Enforcement — CI Gate
```xml
<execution>
    <id>check</id>
    <goals><goal>check</goal></goals>
    <configuration>
        <rules>
            <rule>
                <limits>
                    <limit>
                        <counter>LINE</counter>
                        <value>COVEREDRATIO</value>
                        <minimum>0.80</minimum>  <!-- fail build if < 80% -->
                    </limit>
                </limits>
            </rule>
        </rules>
    </configuration>
</execution>
```

## What Coverage Doesn't Measure
- **Correctness**: Code can be executed but still wrong.
- **Edge cases**: A test can hit a line but with only happy-path inputs.
- **Mutation testing** (PIT) goes further: introduces bugs and checks if your tests catch them.

## Exclusions — What NOT to Count
```java
@Generated  // annotation to exclude from coverage report
public class AutoGeneratedDto { ... }
```

Or in `pom.xml` exclude DTOs, configs, generated code.

## Production Reality
At Zoho, 80% line + 70% branch coverage is the CI gate threshold. JaCoCo reports are published as build artifacts and reviewed in pull request reviews. Teams that drop below threshold are blocked from merging.
